# Plan: Overlay Snooze Button + Duration Setting

## Goal

Add a **snooze button** to the flashcard overlay's top bar (right side, left of the close button). When tapped, it suspends popup display for a user-configured duration, then auto-resumes. The duration is chosen in settings with a 7-step slider: 5 min, 10 min, 30 min, 1 h, 2 h, 6 h, 24 h.

---

## Design Decisions

### Snooze mechanism
- On snooze tap: **stop** the `TimerForegroundService` (same as the existing stop flow), save `pausedUntil = now + snoozeDurationMs` to `SharedPreferences`, and schedule a one-shot `AlarmManager.setExactAndAllowWhileIdle()` firing a new `SnoozeBroadcastReceiver`.
- `SnoozeBroadcastReceiver.onReceive()` checks `pausedUntil != 0L` (guard against manual start/stop cancelling the snooze intent), clears it, and calls `learningServiceManager.startLearningService(intervalMinutes)`.
- `LearningServiceManager.startLearningService()` and `stopLearningService()` both clear `pausedUntil = 0L` so a stale alarm cannot unexpectedly restart after a manual action.
- No new "paused" UI state is added to the main screen (out of scope).

### Settings slider
- 7 discrete options stored as **minutes** in SharedPreferences: `[5, 10, 30, 60, 120, 360, 1440]`.
- Default: **30 min** (index 2).
- The Compose slider maps a float index (0..6) to minutes via a lookup table; displayed as "5 min" / "1 h" etc.

### 2. `service/LearningServiceManager.kt`
- In `startLearningService()`: call `settingsManager.setPausedUntil(0L)` **before** starting the timer.
- In `stopLearningService()`: call `settingsManager.setPausedUntil(0L)` **before** stopping.
- This ensures a pending snooze alarm cannot revive an intentionally-stopped session.

### 3. **New** `service/SnoozeBroadcastReceiver.kt`
```kotlin
@AndroidEntryPoint
class SnoozeBroadcastReceiver : BroadcastReceiver() {
    @Inject lateinit var learningServiceManager: LearningServiceManager
    @Inject lateinit var settingsManager: SettingsRepository

    companion object {
        const val ACTION_RESUME_AFTER_SNOOZE = "com.floflacards.action.RESUME_AFTER_SNOOZE"
    }

    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action != ACTION_RESUME_AFTER_SNOOZE) return
        val pausedUntil = settingsManager.getPausedUntil()
        if (pausedUntil == 0L) return   // was manually overridden, skip
        settingsManager.setPausedUntil(0L)
        learningServiceManager.startLearningService(settingsManager.getIntervalMinutes())
    }
}
```

### 5. `service/OverlayService.kt`
- Add companion constant: `private const val SNOOZE_ALARM_REQUEST_CODE = 2001`

  private fun handleSnooze() {
      val snoozeDurationMs = settingsManager.getSnoozeDurationMinutes() * 60 * 1000L
      settingsManager.setPausedUntil(System.currentTimeMillis() + snoozeDurationMs)
      learningServiceManager.stopLearningService()

  }

  - Calls `setExactAndAllowWhileIdle()` (API ≥ 23) or `setExact()` (older).
- In `showOverlay()`, pass `onSnooze = { handleSnooze() }` to `OverlayContent()`.
- Add necessary imports: `android.app.AlarmManager`, `android.app.PendingIntent`.

## Call Graph After Changes

```
Overlay snooze button tap
  → OverlayComponents.onSnooze()
    → OverlayService.handleSnooze()
      → settingsManager.setPausedUntil(now + duration)
      → learningServiceManager.stopLearningService()   [sets pausedUntil=0 AFTER we set it — BUG risk]
      → scheduleSnoozeResume(durationMs)               [AlarmManager]
      → closeOverlay()

// NOTE: stopLearningService() clears pausedUntil — so we must set it AFTER calling stop.
// Order in handleSnooze():
//   1. learningServiceManager.stopLearningService()   ← clears pausedUntil
//   2. settingsManager.setPausedUntil(now + duration) ← re-set after
//   3. scheduleSnoozeResume()
//   4. closeOverlay()

SnoozeBroadcastReceiver.onReceive()
  → pausedUntil = settingsManager.getPausedUntil()
  → if (pausedUntil == 0L) return   // manual override happened
  → settingsManager.setPausedUntil(0L)
  → learningServiceManager.startLearningService(intervalMinutes)
```
